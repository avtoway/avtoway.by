# АВТОWAY — Архитектура и стратегия проекта

## Философия
**Главная страница** — лицо компании и моё лицо. Это продающая страница, вызывающая доверие. 
Всё строится вокруг видео на YouTube — я снимаю про авто и горю этим.
Каждая услуга — отдельный мини-проект внутри основного, со своим суббрендом (АВТОWAY.УСЛУГА).

---

## 1. Структура сайта

### Навигация (хедер)
- **Главная** (логотип + кнопка)
- **О нас** (отдельная страница, не скролл)
- **Видео** (удалить из меню, будет только на главной)
- **Услуги** (выпадающее меню со всеми услугами)
- **Партнёры** (отдельная страница)
- **Контакты** (отдельная страница)

### Страницы
| Страница | Тип | URL |
|----------|-----|-----|
| Главная | landing page | `/` |
| О нас | информация + ссылки на все ресурсы | `/about` |
| Услуги | landing page | `/services` |
| Партнёры | карточки → детальная страница | `/partners`, `/partners/[id]` |
| Контакты | форма связи, соцсети | `/contacts` |

### Услуги (выпадающее меню)

| Услуга | Суббренд | Тип страницы | URL |
|--------|-----------|-------------|-----|
| Аренда авто | АВТОWAY.АРЕНДА | каталог + бронь | `/rent` |
| Таксопарк | АВТОWAY.ТАКСОПАРК | landing page | `/taxi` |
| Продажа авто | АВТОWAY.АВТО | каталог с карточками | `/cars` |
| Эвакуатор | АВТОWAY.ЭВАКУАТОР | landing page | `/tow` |
| Автоподбор | АВТОWAY.ПОДБОР | landing + услуги | `/inspection` |
| Автопригон | АВТОWAY.ПРИГОН | landing page | `/import` |
| Автосервис | АВТОWAY.СЕРВИС | landing page | `/service` |
| Детейлинг | АВТОWAY.ДЕТЕЙЛИНГ | landing page | `/detailing` |

## 2. Архитектурные требования

### Паттерны
- **Feature-Sliced Design** — разделение по модулям/фичам
- **Repository Pattern** — слой абстракции над БД
- **Server Components** — по умолчанию, Client Components только где нужно
- **API Routes (Next.js App Router)** — для админки и интеграций
- **Middleware** — для защиты админки и ролей

### Структура папок (src/)
```
src/
├── app/                    # Next.js App Router
│   ├── (main)/             # Публичная часть
│   │   ├── page.tsx        # Главная
│   │   ├── about/
│   │   ├── contacts/
│   │   ├── partners/
│   │   ├── services/
│   │   └── ...услуги
│   ├── admin/              # Админка
│   │   ├── login/
│   │   └── dashboard/
│   └── api/                # API routes
├── components/
│   ├── ui/                 # Переиспользуемые UI (button, input, card...)
│   ├── layout/             # Header, Footer, Sidebar
│   └── features/           # Feature-specific components
├── lib/
│   ├── db/                 # Prisma client, migrations
│   ├── youtube/            # YouTube API
│   ├── social/             # Instagram, Telegram, VK, Rutube
│   ├── maps/               # Яндекс.Карты
│   └── ads/                # Google Adsense / Яндекс
├── models/                 # TypeScript types/interfaces
├── hooks/                  # React hooks
└── utils/                  # Helpers
```

## 3. База данных

### Основные сущности (Prisma)

```
User           — пользователи системы (админы, менеджеры)
Role           — роли (Admin, Editor, RentManager, Partner...)
Service        — услуги (справочник)
Partner        — партнёры (название, описание, фото, ссылки, контакты)
PartnerDetail  — детальная информация партнёра

# Аренда
RentalCar      — автомобили для аренды (марка, модель, фото, статус)
RentalOrder    — заказы аренды (клиент, даты, статус, цена)
RentalClient   — клиенты аренды (ФИО, телефон, паспортные данные)

# Продажа авто
CarForSale     — авто на продажу (марка, модель, описание, цена, статус)
CarChecklist   — чеклист осмотра авто
CarPhoto       — фотографии авто
CarValuation   — оценка (цена, средняя, высокая, низкая)

# Автоподбор
ClientOrder    — заказы на подбор
InspectionReport — отчёт проверки
InspectionService — услуги подбора (разовая, эксперт, под ключ)

# Партнёры
PartnerService — услуги партнёров

# Общие
Review         — отзывы (услуга, автор, фото, текст, рейтинг, source)
ContactRequest — заявки с формы обратной связи
Setting        — настройки сайта
```

### Роли и доступ
| Роль | Доступ |
|------|--------|
| Admin | Всё |
| Editor | Контент (страницы, услуги) |
| RentManager | Управление арендой |
| SalesManager | Управление продажей авто |
| Partner | Свой раздел партнёра |

## 6. Ключевые решения

- Никакой регистрации пользователей (кроме админов)
- Доступ по ролям через админ-панель
- Каждая услуга — модуль, который можно включить/отключить
- Суббренды (АВТОWAY.УСЛУГА) — через мета-теги и логотип
- Все данные в PostgreSQL через Prisma
- API — Next.js API routes (App Router)
- Деплой — Vercel + Neon (PostgreSQL)
